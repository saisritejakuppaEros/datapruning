# SemDeDup for CLIP Embeddings (LAION Aesthetic)

Run SemDeDup on CLIP embeddings generated by `clip_embeds_gen.py`. No SLURM required.

## Prerequisites

1. **Conda env** with SemDeDup dependencies (including faiss):
   ```bash
   conda env create -n semdedup --file semdedup_conda_env.yml
   conda activate semdedup
   ```

2. **CLIP embeddings** already generated in parquet format (from `clip_embeds_gen.py`).

3. **Base path** (used below): `/mnt/data0/teja/data_pruning`

## Pipeline Overview

1. **Convert parquet → memmap** (`parquet_to_memmap.py`)
2. **K-means clustering** (`compute_centroids`)
3. **Sort clusters** (`assign_and_sort_clusters`)
4. **SemDeDup** (identify semantic duplicates)
5. **Extract pruned paths** (output list of kept image paths)

## Step-by-Step

### Step 0: Convert parquet to memmap

```bash
cd /mnt/data0/teja/data_pruning

python codes/SemDeDup/parquet_to_memmap.py \
  --clip_embeds_dir output/clip_embeds \
  --output_dir output/semdedup_embeds \
  --dataset_path /datasets/ai-core-object/d-gpu-06097851-2053-4b67-8400-b5d404c04261/teja/internet_dataset/laionasthetic_v2 \
  -j 8
```

- `-j` / `--workers`: parallel workers for reading parquet files (default: 1).
- Skips invalid/NaN embeddings and missing parquet files automatically.
- Output: `embeddings.npy`, `paths.npy`, `dataset_info.json` in `output/semdedup_embeds/`.

### Step 1–5: Run SemDeDup (local)

```bash
cd /mnt/data0/teja/data_pruning/codes/SemDeDup

python run_semdedup_local.py \
  --embeddings_dir /mnt/data0/teja/data_pruning/output/semdedup_embeds \
  --eps_list 0.06 0.07 \
  --ncentroids 50000
```

- `--eps_list`: Lower ε = more aggressive pruning (see [paper](https://arxiv.org/abs/2303.09540)).
- Output: `kept_paths_eps0.06.txt` (or `kept_paths_eps<eps>.txt`).

### Optional: Resolve paths to full image locations

```bash
python resolve_dedup_paths.py \
  /mnt/data0/teja/data_pruning/output/semdedup_embeds/kept_paths_eps0.06.txt \
  --dataset_path /datasets/ai-core-object/d-gpu-06097851-2053-4b67-8400-b5d404c04261/teja/internet_dataset/laionasthetic_v2 \
  --output_txt kept_paths_resolved.txt
```

Output format: `{dataset_path}/{rel_tar_path}::{member_path}` — use with tarfile to extract images.

### Useful flags for `run_semdedup_local.py`

| Flag | Description |
|------|-------------|
| `--skip_clustering` | Skip steps 2–3 if already run |
| `--skip_semdedup` | Skip step 4 if already run |
| `--extract_only` | Only run step 5 (extract paths) |
| `--eps_for_extract` | Which ε to use when extracting (default: first in eps_list) |
| `--output_txt` | Output file for kept paths |

## Path format

- **Stored path**: `rel_tar_path|path` (e.g. `split_000/00000.tar|00000/000000534.jpg`)
- **Full tar path**: `{dataset_path}/{rel_tar_path}`
- **Member in tar**: `path` (parquet `path` column = tar member name)
